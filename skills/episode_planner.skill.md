# 單集規劃師技能 (Episode Planner)

> 📚 **專業參考**
> - Pamela Douglas《Writing the TV Drama Series》
> - Daniel Calvisi《Story Maps: TV Drama》
> - Netflix 劇本開發指南

---

## 一、核心任務

將故事大綱的**單集概要**擴展為**詳細的單集規劃**，包括：
- 場景分解
- 情節節拍
- 情緒弧線
- 視覺提示

---

## 二、批次處理

每次處理 **10集**，輸出約 **5000字**。

輸入：
- 第N-M集的簡要概要
- 人物關係圖
- 世界觀設定
- 前面已完成的集數概要（保持連貫）

輸出：
- 10集詳細規劃

---

## 三、單集規劃結構 (每集約500字)

### 場景分解
```
場景1: [地點] - [時間] - [在場角色]
  └── 事件：發生什麼
  └── 目的：推動什麼

場景2: ...
場景3: ...
（每集3-8個場景，依時長調整）
```

### 情緒曲線
```
開頭: [情緒狀態] (如：平靜、緊張、興奮)
中段: [情緒轉變] (如：從希望到失望)
結尾: [情緒落點] (如：懸念、震驚、溫暖)
```

### 關鍵對白提示
```
- 場景X 需要一句金句：主題是___
- 場景Y 需要衝突對白：關於___
```

### 視覺提示
```
- 標誌性畫面：描述一個印象深刻的鏡頭
- 色調建議：暖/冷/對比
- 特殊視覺：下雨/夜景/人群等
```

---

## 四、輸出格式

```json
{
  "batchRange": "EP01-EP10",
  "episodes": [
    {
      "ep": 1,
      "title": "標題",
      "duration": "3分鐘",
      "scenes": [
        {
          "num": 1,
          "location": "地點",
          "time": "日/夜",
          "characters": ["角色A", "角色B"],
          "event": "發生什麼",
          "purpose": "推動什麼"
        }
      ],
      "emotionCurve": {
        "start": "平靜",
        "middle": "緊張",
        "end": "懸念"
      },
      "keyDialogue": [
        "場景2需要表達___的台詞"
      ],
      "visualHints": {
        "keyShot": "描述標誌性畫面",
        "colorTone": "冷色調",
        "special": "雨夜"
      },
      "hook": "結尾鉤子是什麼"
    }
  ]
}
```

---

## 五、連貫性檢查

每批次生成前，必須考慮：
- ✅ 上一批最後一集的結尾狀態
- ✅ 人物關係的變化
- ✅ 伏筆的埋設與回收
- ✅ 情緒節奏的起伏

---

## 六、時長與場景對照

| 每集時長 | 建議場景數 | 每場景平均 |
|----------|-----------|-----------|
| 2分鐘 | 2-4場 | 30-60秒 |
| 3分鐘 | 3-5場 | 36-60秒 |
| 5分鐘 | 4-7場 | 43-75秒 |
| 10分鐘 | 8-12場 | 50-75秒 |

---

## 七、禁止事項

❌ 不要重複上一集的情節
❌ 不要讓角色突然性格大變（除非有鋪墊）
❌ 不要忘記伏筆
❌ 不要每集情緒都一樣
❌ 不要寫「下一集預告」
